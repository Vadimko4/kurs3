# Проект "KURS3"

## Описание:

Проект является курсовой работой к третьему модулю обучающего курса Skypro "Профессия Python-разработчик".
Разрабатыватся приложение для анализа транзакций, которые находятся в Excel-файле. 
Приложение должно генерировать JSON-данные для веб-страниц, а также предоставлять другие сервисы.

Реализованы следующие задачи по категориям:
1. Веб-страницы: Главная
2. Сервисы: Выгодные категории повышенного кешбэка
3. Отчеты: Траты по категории

1. get_views_json: основная функция модуля views. Принимает на вход дату запроса.
    Получает из xlsx файла список транзакций.
    Отфильтровывает по статусу ОК, по дате от начала месяца до request_date + "23:59:59"
    Анализирует итоговый список.
    По результатам анализа возвращает json в который запакован словарь.
    В словаре следующие ключи:
    "greeting" - приветствие, адекватно времени обращения к функции

    "cards" - информация по картам, список словарей вида:
    "last_digits": "5814",
      "total_spent": 1262.00,
      "cashback": 12.62

    "top_transactions" - 5 самых больших расходов за период в виде:
    "date": "20.12.2021",
    "amount": 829.00,
    "category": "Супермаркеты",
    "description": "Лента"

    "currency_rates" - курсы валют (берутся из user_settings.json файла) к рублю список словарей вида:
    "currency": "USD",
    "rate": 73.21

    "stock_prices" - стоимость акций (берутся из user_settings.json файла) в рублях в виде:
    "stock": "AAPL",
    "price": 150.12
2. get_profitable_cashback_categories: основная функция модуля services - сервис 
    «Выгодные категории повышенного кешбэка» позволяет проанализировать, какие категории были наиболее выгодными
    для выбора в качестве категорий повышенного кешбэка.
    Функция сервиса принимает год, месяц для расчета и транзакции в формате списка словарей.
    Возвращает JSON-ответ с анализом, сколько на каждой категории можно заработать кешбэка в указанном месяце года.
    Формат выходных данных:
    {
        "Категория 1": 1000,
        "Категория 2": 2000,
        "Категория 3": 500
    }
3. spending_by_category: основная функция модуля reports - возвращает из входного дата фрейм с банковскими транзакциями 
    дата фрейм с банковскими транзакциями,
    которые кассаются данной категории в параметре category и за 3 предыдущих месяца до передаваемой в параметре
    date даты; если параметр не передан. то отсчёт производится от текущего момента
4. Большинство вспомогательных функций прописаны в модуле utils
5. Модуль external_api отвечает за взаимодействие программы с API. Соответствующие ключи считываются из переменных 
   состояния файла .env Шаблон для него .env.example размещён в репозитории. 
6. Все основные моменты работы программы логируются при помощи логгеров, созданных в модуле logger
7. Модуль decorators реализует функцию-декоратор, которая записывает результат работы функции spending_by_category,
   который является дата фреймом пандас - в xlsx файл, который находится в папке reports
8. Файл main связывает логику проекта в единое целое - общение программы с пользователем.

Все функции проекта оттестированы в различных условиях: в штатных и нештатных, когда в параметрах всякая неожидаемая ерунда,
либо когда генераторы ничего не возвращают, за отсутствием подходящей под условия запроса информации. Дабы программа не падала с ошибками.

## Установка:

1. Клонируйте репозиторий:
```
git clonegit@github.com:Vadimko4/kurs3.git
```
или
```
git clone https://github.com/Vadimko4/kurs3.git
```

2. Установите зависимости:
```
poetry install
```
3. Ознакомьтесь с инструкцией в файле .env.example и донастройте окружение (введите свои API_KEY)

## Использование:

Запустите на выполнение файл main.py
